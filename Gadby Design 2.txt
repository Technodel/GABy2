# GABy (Galaxy AI Bot) – Comprehensive Re-Design Plan

## 1. Executive Summary

### Goal

To take the functional core of `hotovo/aider-desk` and transform its appearance, user experience, and business model.

### Key Performance Indicators (for the Agent)

* **Tool Name:** Universally changed to **GABy**. All references to Aider/Aider Desk removed.
* **Target Audience:** Non-technical or low-experience coding individuals.
* **UI Aesthetic:** Totally minimal, friendly, and non-intimidating (think minimal markdown editor + polite status bar).
* **System Model:** Monetized per user. Admin adds users with a balance; usage decreases the balance based on a mark-up formula.
* **Security:** Hides all AI model names, API keys, and technical CLI instructions from end-users.

## 3. Backend & Architectural Modifications (VPS Setup)

Because `aider-desk` was primarily a desktop application, the core adaptation involves migrating its logic into a multi-user, web-accessible service that can run on a Linux VPS.

### 3.1. User Management & Authentication System

Create a persistent database (e.g., PostgreSQL or SQLite) with the following schema:

#### Database Schema

* **`system_config` (Table)**
* `id` (int, PK)
* `admin_password_hash` (string) – Default hash for "galaxy password 301088".
* `api_keys` (json) – Stores Anthropic/OpenAI keys.
* `token_pricing_formula` (json) – e.g., `{ "markup_factor": 2.0 }`.
* `global_max_tokens` (int)


* **`users` (Table)**
* `id` (int, PK)
* `username` (string, unique)
* `password_hash` (string)
* `balance_cents` (int) – Store currency in cents to avoid precision errors.
* `is_admin` (boolean)


* **`usage_logs` (Table)**
* `id` (int, PK)
* `user_id` (int, FK)
* `project_id` (string)
* `tokens_input` (int)
* `tokens_output` (int)
* `model_cost_cents` (int)
* `charged_user_cents` (int)



### 3.2. Authorization (Gatekeeping)

Create backend API endpoints with specific authorization requirements:

* `/api/admin/*` $\rightarrow$ Requires valid Admin Session or specific Admin Password header ("galaxy password 301088").
* `/api/user/*` $\rightarrow$ Requires a standard logged-in user session.
* `GET /` $\rightarrow$ Serves the login page.

### 3.3. Balancing & Pricing Logic

#### The Formula

$$Charged Price = (Total Token Cost) \times markup\_factor$$

#### Implementation Details

Hook into the Vercel AI SDK or Aider's usage parsing mechanism. When GABy receives a final model response containing token usage counts:

1. Lookup the standard raw token costs for the model used (hidden from user).
2. Calculate raw `model_cost_cents`.
3. Calculate `charged_user_cents` using the markup factor from `system_config`.
4. Lookup the user's current `balance_cents`.
5. If `balance_cents` < `charged_user_cents`, stop the current operation and inform the user that their celestial fuel has run out (polite message).
6. Otherwise, decrease user balance and log the transaction to `usage_logs`.

### 5.3. Task Execution Display Adaption (Suppressing Raw Commands)

This is the most critical logic change for non-experienced users. We must stop showing technical CLI commands and show friendly descriptions derived from what those tools are actually *doing*.

#### Implementation Requirement

Modify the component that streams the `Executing tool...` updates. Use pattern matching on Aider's output to map technical commands to friendly descriptions.

| Aider Tool/Command Pattern | Suppress Raw CLI? | friendly/Fun Description to Show Instead |
| --- | --- | --- |
| `Run commandcd D:\Projects\...; node scripts/...` | **YES** | "Executing a bad staged file removal script from your `scripts/` folder." |
| `Run commandls ...` | **YES** | "Gazing at the files in this folder to understand what we have." |
| `Executing tool (grep)...` | **YES** | "Scanning files for that specific text pattern you mentioned!" |
| `Applying git commit...` | **YES** | "Recording our changes into your project's cosmic history (Git commit)." |
| `Executing tool (read_file)...` | **YES** | "Devouring the contents of `{filename}` to plan our edits." |

## 10. Development & VPS Deployment Spec

### Technology Stack adapted for VPS

* **Frontend:** React/TypeScript web interface (removed desktop dependencies).
* **Backend:** Node.js service (adapt current Vercel AI logic to a long-running Express service).
* **Database:** SQLite (easiest setup for a simple VPS) or PostgreSQL.
* **Deployment:** Containerized (Docker).

### Deployment Implementation Steps for the Agent

1.  Adapt current desktop codebase to be browser-accessible.
2. **Initialize Database Schema** (Step 3.1).
3. **Implement Admin Panel** (Step 8) with primary key storage.
4. **Implement Renaming** (Step 2).
5. **Build Final Re-designed Frontend** (Steps 4-7, 9).
6. **Create Dockerfile and docker-compose.yml.**
* Service 1: GABy Web App (Node.js).

7. **Setup VPS:**
* Install Docker & docker-compose.
* Configure domain name (DNS A-record).

8. **Setup Script:** Create a simple shell script (`install_gaby.sh`) that initializes the DB and sets the initial admin password from environment variables.
